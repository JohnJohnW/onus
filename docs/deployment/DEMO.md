# Onus - free demo deployment

A free, public **demo** of Onus: front end on Vercel, engine on Render, database on Neon
(all US free tiers). This is for evaluation only - it is US-hosted, so **do not enter real
client data**. The app shows a demo banner and a `/hosting` page that disclose this and let
visitors register interest in a properly Australian-hosted deployment (see the main
[README](../../README.md) data-residency section).

> For a real, compliant deployment in Australia, follow [README.md](README.md) instead.

You'll create three free accounts (no credit card needed on these tiers). I can't do the
account / login / terms / secret-entry steps for you, but the repo is set up so each is a
few clicks.

Order matters: **database -> engine -> front end** (each needs the previous one's URL).

---

## 1. Database - Neon (free Postgres)

1. Sign up at neon.tech and create a project (any US region).
2. Open the project's **SQL Editor** and run (pick a strong password):

   ```sql
   CREATE EXTENSION IF NOT EXISTS vector;
   CREATE ROLE onus_app LOGIN PASSWORD 'CHOOSE_A_STRONG_PASSWORD'
     NOSUPERUSER NOCREATEDB NOCREATEROLE NOBYPASSRLS;
   GRANT USAGE, CREATE ON SCHEMA public TO onus_app;
   ```

   (Row-level security still applies to the app even via the owner role because the
   migrations `FORCE` it, but the dedicated `onus_app` role keeps least-privilege intact.)
3. From the Neon dashboard, copy two connection strings (psql/URI form):
   - the **owner** connection string -> this is `ALEMBIC_DATABASE_URL`
   - the same string but with user `onus_app` and the password you chose ->
     `DATABASE_URL` (and `ONUS_APP_PASSWORD` = that password)

## 2. Engine - Render (free, Docker)

1. Sign up at render.com and connect your GitHub (this repo).
2. **New + > Blueprint**, pick the repo. It reads `render.yaml` and creates `onus-engine`.
3. Set the secret env vars when prompted (or under the service's Environment tab):
   - `DATABASE_URL` = the Neon `onus_app` string from step 1
   - `ALEMBIC_DATABASE_URL` = the Neon owner string
   - `ONUS_APP_PASSWORD` = the password you chose for `onus_app`
   - `ANTHROPIC_API_KEY` = your key (optional; needed for AI drafting)
   - `JWT_SECRET` is auto-generated by Render.
4. Deploy. On start the engine runs `alembic upgrade head` then serves. When it's live,
   copy its URL, e.g. `https://onus-engine.onrender.com`. Check `/health` returns ok.
   (Free instances sleep after ~15 min idle; the first request then takes ~30-60s.)

## 3. Front end - Vercel (free, Next.js)

1. Sign up at vercel.com, **Add New > Project**, import this repo, authorize GitHub.
2. Set **Root Directory** to `web` (the Next.js app lives there). Framework auto-detects
   as Next.js.
3. Add Environment Variables:

   | Name | Value |
   |---|---|
   | `ENGINE_INTERNAL_URL` | your Render engine URL (e.g. `https://onus-engine.onrender.com`) |
   | `NEXTAUTH_URL` | your Vercel URL (e.g. `https://onus-demo.vercel.app`) |
   | `NEXTAUTH_SECRET` | a long random value - run `openssl rand -hex 48` |
   | `NEXT_PUBLIC_DEMO` | `true` |

4. Deploy. If your final Vercel domain differs from what you set in `NEXTAUTH_URL`, update
   it and redeploy.

## Verify

- Visit the Vercel URL: the landing page loads with the amber **demo banner**.
- `/hosting` shows the trade-offs and the expression-of-interest form.
- Sign up a firm, complete onboarding, and confirm the dashboard populates.
- Expressions of interest land in the `demo_eois` table (read via Neon's SQL editor:
  `SELECT * FROM demo_eois;`).

## Notes / limits (demo only)

- US-hosted -> not suitable for real client data (the banner says so).
- Render free: sleeps when idle; ephemeral disk, so uploaded documents do not persist
  across restarts.
- To move to a compliant Australian deployment later, see [README.md](README.md) - the
  same images, pointed at Australian-region infrastructure.
